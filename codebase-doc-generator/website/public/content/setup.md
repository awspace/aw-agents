# Setup Guide

This guide will help you set up and run the documentation website locally.

## Prerequisites

Before you begin, ensure you have the following installed on your system:

- **Node.js** (version 18 or higher)
- **npm** or **yarn** package manager

## Installation

1. Navigate to the website directory:
   ```bash
   cd codebase-doc-generator/website
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

## Development

To start the development server with hot reload:

```bash
npm run dev
```

The application will be available at `http://localhost:3421`

## Build for Production

To create an optimized production build:

```bash
npm run build
```

The built files will be in the `dist` directory, ready to be deployed to any static hosting service.

## Preview Production Build

To preview the production build locally:

```bash
npm run preview
```

## Content Generation

The documentation content is generated by the `codebase-doc-generator` agent. To generate content for your own project:

1. Run the agent on your target codebase
2. The agent will automatically place generated content files in `public/content/`
3. Rebuild the website to include the new content

## Deployment

The static build can be deployed to any static hosting service, including:

- **Vercel**
- **Netlify**
- **GitHub Pages**
- **Cloudflare Pages**
- **AWS S3 + CloudFront**
- Any web server that can serve static files

### Deploying to GitHub Pages

1. Build the project:
   ```bash
   npm run build
   ```

2. Push the `dist` directory to your `gh-pages` branch:
   ```bash
   npm install -g gh-pages
   gh-pages -d dist
   ```

## Customization

### Changing the Port

Edit the `vite.config.ts` file and modify the `server.port` property:

```typescript
export default defineConfig({
  // ...
  server: {
    port: 3421, // Change to your preferred port
    open: true,
  },
})
```

### Adding Custom Styling

Edit `src/index.css` to add custom styles or modify existing ones. The project uses Tailwind CSS, so you can also customize the Tailwind configuration in `tailwind.config.js`.

### Adding New Sections

1. Create a new page component in `src/pages/`
2. Add the route to `src/App.tsx`
3. Add a navigation item to `src/components/Sidebar.tsx`
4. Update the content schema and loader functions as needed

## Troubleshooting

### Dependencies won't install

- Clear your npm cache: `npm cache clean --force`
- Delete `node_modules` and `package-lock.json` and reinstall
- Try using yarn instead: `yarn install`

### Build fails

- Ensure you're using Node.js 18 or higher
- Check for TypeScript errors: `npm run typecheck`
- Verify all content files in `public/content/` are valid JSON

### Content won't load

- Check the browser console for error messages
- Ensure all required content files exist in `public/content/`
- Verify JSON files are valid and follow the schema
- Check the network tab to see if files are being loaded correctly

### Diagrams won't render

- Ensure Mermaid syntax is valid
- Check the browser console for Mermaid error messages
- Simplify complex diagrams if they're causing rendering issues

## Getting Help

If you encounter issues or have questions about using the codebase-doc-generator:

1. Check the agent documentation in `codebase-doc-generator.md`
2. Review the example content files in `public/content/` for schema reference
3. Ensure all files follow the specified JSON schemas
